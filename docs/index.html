Why

LLM subscriptions meter you in invisible windows — Claude's 5-hour session and weekly caps, Codex's 5-hour and weekly limits. You find out you've burned the window when the wall hits, usually mid-task. limitbar keeps the current windows visible: in your terminal, or as a tiny translucent click-through bar floating above your editor.

No login. Ever.

limitbar never asks you to authenticate and never talks to a server on its own. Your tools already wrote everything it needs onto your own disk — limitbar just reads it.

Official numbers where they exist

Codex logs its real used-percentages locally; limitbar reports them verbatim. For Claude Code it counts exact tokens from your transcripts, deduplicated per billed response.

Click-through overlay

A frameless, transparent, always-on-top bar that ignores your mouse. Park it over your editor's title bar and forget it's a window.

Install

Debian/Ubuntu (apt)

curl -fsSL https://pub-85a506db1b4547ba970ecf62c38f89b4.r2.dev/rocky-oss.gpg \
  | sudo tee /usr/share/keyrings/rocky-oss.gpg >/dev/null
echo "deb [signed-by=/usr/share/keyrings/rocky-oss.gpg] https://pub-85a506db1b4547ba970ecf62c38f89b4.r2.dev stable main" \
  | sudo tee /etc/apt/sources.list.d/rocky-oss.list
sudo apt update && sudo apt install limitbar

macOS / Linux (Homebrew)

brew tap i-rocky/tap
brew install limitbar

Windows (Scoop)

scoop bucket add rocky https://github.com/i-rocky/scoop-bucket
scoop install limitbar

From source (overlay included)

cargo install --git https://github.com/i-rocky/limitbar --features overlay

Prebuilt binaries for Linux (x86_64/aarch64), macOS (Intel/Apple Silicon), and Windows are on GitHub Releases, with SHA256 checksums. All include the overlay.

Usage

There is no setup and no account. Install, then:

CommandWhat it does
limitbarprint the current windows once and exit
limitbar -w 10live-refresh in the terminal every 10 seconds
limitbar --overlayfloating always-on-top click-through bar (build with --features overlay)

Optional budgets

Anthropic doesn't publish the token caps behind Claude's windows, so percentage gauges for claude-code appear once you set budgets calibrated to your plan in ~/.config/limitbar/config.toml. Without them you still get exact totals and reset times.

[budgets.claude-code]
five_hour_tokens = 500000000
weekly_tokens   = 3000000000

Providers

ProviderStatusData source
claude-code✅ shippedlocal transcripts (~/.claude/projects) — exact tokens, fully offline
codex✅ shippedlocal session logs (~/.codex/sessions) — official used-percentages
cursorplannedCursor's stored session + usage API; same zero-prompt rule
antigravityplannedlocal app data, format being mapped

Tools that aren't installed are silently skipped. Adding a provider is one file implementing the Provider trait — read what the app already wrote, return windows.
